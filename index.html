Week 3

  • Forms, CSS, Positioning, Flexboxes
  • Forms
  • CSS Inheritance, Specificity, Cascade and Importance
    • Inheritance - Styles inherited from parent elements.
    • Specificity - Which CSS rule takes priority.
    • Cascade - Style declarations read from top to bottom.
    • Importance - !Important overrides source order and specificity.
  • Media Queries - modify the CSS depending on device type/viewport size.
  • Positioning
    • Relative - Moves elements around relative to where they appear in a page.
    • Absolute - Offset relative to the entire browser window instead of original positioning.
    • Fixed - Absolute except coordinates are relative to the browser window.
    • Z-Index - Controls the depth of elements on a page.
  • Flexboxes - Containers to flex items that adapt layout to suit contents.

Week 5

  • Introduction to Javascript basics
  • Javascript Objects - group together a set of variables/functions to create a model.
  • Access the object through calling the object and their property or method name.
  • Update an object through calling the object and their propery name, and give it a new value.
  • Delete a property of an object through using keyword "delete".
  • Literal Objects has no constructors and cannot be inherited,
    but Function-based objects has a constructor and can be inherited.
  • Var/Let/Const
    • Var - Able to be hoisted/can be called regardless of where the actual declaration appear.
      •  
    • Let - Not hoisted to the top of an enclosing block.
    • Block-scoped, only existing within the brackets it is contained in.
      •  
    • Const - Constants, values cannot be changed once set.

Week 6

  • More Javascript basics
  • DOM
    • A model of a loaded page - called the DOM tree.
    • Document Node - Represents the entire page document.
    • Element Node - Elements that describe the structure of a HTML page.
    • Attribute Node - Opening tags of elements with attributes like Classes and IDs.
    • Text Node - On accessing the element node, you can modify the text within.
    • Can access elements through using getElementById(id)/getElementsByClassName(class) and etc...
    • The .textContent property allows getting of content of an element or modifying of content.
    • the .innerHTML property is the same, but HTML markup is allowed.

Week 7

  • DOM Events and Local Storage
  • DOM Events
    • Event Listeners - Adding an event listener to a DOM element node.
    • Example: button.addEventListener('click', function(){}, false)
      •  
    • Event Objects - Every time an event happens, the event object contains helpful data about it.
    • Changing Default Behavior - preventDefault() prevents the default action of an event from triggering.
    • Changing Default Behavior - stopPropagation() stops the events from propagating to parent DOM elements.
  • Local Storage
    • Based on named key and value pairs.
    • Store data based on a named key, then retrieve the data with the same key.
    • Data can be any type supported by JavaScript, but is stored as a string.
    • Have to use functions like parseInt() to change the data to the expected type.
    • Local and Session storage - used in place of cookies using the localStorage and sessionStorage objects.
    • Stores up to 5MB of data per domain.

Week 9

  • Bootstrap
  • A powerful, extensible and feature-packed frontend toolkit.
  • Developed by developers that wanted a faster way to create User Interfaces(UI) and resolve various issues.
  • It has a flexbox grid that has device breakpoints to accomodate for different media sizes and helps to create a layout for a responsive website.
  • It has many components for many parts of a website like forms and navbars to create responsive designs.
  • It can also make video embeds responsive to adjust its size based on the device.
 

Week 12

  • NoSQL/RESTDB
  • NoSQL
    • Most commonly known as "Not only SQL database", which provides storage and retrieval of unstructured data.
    • Can handle an immense amount of data and has a dynamic schema.
    • No specific query language, almost no relationships but stored in a format of collections and documents.
    • Create and use RESTDB databases to use NoSQL.
  • Use API requests PUT / PATCH / POST / DELTE / GET
 

Week 14

  • Lottie
  • A way to perform animations and allow more interactivity to a site.
  • JSON-based animation file format that allows designers to ship animations on any platform.
  • Small files that works on any device and can scale up/down without pixelation.
  • Extremely lightweighted Animation and Design as it is just code - reducing disk space used and increasing download speed.
